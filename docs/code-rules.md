## Code rules:

- respect the project structure and naming rules
- don't use the private keyword in classes
- use the unserscore notation (\_camelCase) for private fields defined in a class
- write the names of methods, classes, interfaces, constants and public fields in PascalCase
- variable names are written camelCase
- a line of code should not exceed 140 characters
- if a method with parameters exceeds 140 characters, put each parameter on a new line (even the first parameter)
- if the name of the class and interface it implements exceeds 140 characters, put the semicolon and the interface on a new line
- do not use shortcuts in names. You can only use it if it is a commonly known shortcut, e.g. EF - EntityFramework.
- interface names start with I
